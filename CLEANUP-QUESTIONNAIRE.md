# Опрос: что удалить для облегчения проекта (статики больше не нужна)

Ниже перечислены кандидаты на удаление. **Ничего не удалено автоматически** — отметьте то, с чем согласны удалить, и выполните удаление сами или попросите выполнить по пунктам.

**Как помечать:** ставьте **☑** вместо ☐ в колонке «Удалить?» для пунктов, которые согласны удалить.

## 2. Статические страницы артистов (папка `artists/`)

Отдельные HTML-страницы артистов; контент теперь в CPT `artist` и шаблоне single-artist.php.

| Удалить? | Что | Описание |
|----------|-----|----------|
| ☐ | `artists/crylove/artist.html` | Статическая страница артиста CRYLOVE. |
| ☐ | `artists/dimma-urih/artist.html` | Статическая страница артиста DIMMA URIH. |
| ☐ | `artists/dsprite/artist.html` | Статическая страница артиста DSPRITE. |
| ☐ | `artists/net-vremeni-ob-yasnyat/artist.html` | Статическая страница артиста НЕТ ВРЕМЕНИ ОБЪЯСНЯТЬ. |
| ☐ | `artists/nxn/artist.html` | Статическая страница артиста NXN. |

**Важно:** в папках `artists/*/` могут лежать не только HTML, но и медиа (фото, обложки). Перед удалением проверьте: если медиа для артистов подгружаются из темы (`wp-content/themes/oor-theme/public/assets/artists/`), то удалять можно только `.html` или целиком папку, если медиа продублированы в теме.

**Рекомендация:** удалить всю папку `artists/` только если медиа артистов уже есть в теме (oor-theme/public/assets/artists/).

## 3. Исходники статики: папка `src/`

Содержит исходные CSS и JS, которые уже скопированы в тему: `wp-content/themes/oor-theme/assets/css/` и `assets/js/`. В WordPress подключаются файлы из темы, не из `src/`.

| Удалить? | Что | Описание |
|----------|-----|----------|
| ☐ | `src/css/` | Исходники CSS (reset, tokens, base, grid, layout, fonts, utilities, slider, scrollbar, animations, components, cursor). |
| ☐ | `src/js/` | Исходники JS (main, config, cursor, slider, mobile-menu и т.д.). |
| ☐ | `src/body-classes.php`, `src/enqueue.php`, `src/footer.php`, `src/header.php` | PHP из корня src/ — логика перенесена в тему (inc/, header.php, footer.php). |
| ☐ | `src/page-*.php` | Страницы в src/ — в теме есть свои page-*.php. |

**Примечание:** в теме в `config.js` и прелоадере есть ссылки на `/src/css` и `/src/js` как fallback; при работе через WordPress используются `window.oorPaths` от темы. После удаления `src/` эти fallback-пути не будут работать при статическом просмотре — это нормально, если статику вы больше не используете.

**Рекомендация:** удалить всю папку `src/` после того, как убедитесь, что правки делаете только в файлах темы (oor-theme/assets/).

## 4. Корневая папка `public/` (медиа: картинки, шрифты, видео)

Около **774 файлов** (PNG, AVIF, WebP, шрифты, видео и т.д.). Тема обращается к ресурсам так: `get_template_directory_uri() . '/public/assets'` и `...'/public/fonts'`, то есть ожидает папку **public внутри темы**: `wp-content/themes/oor-theme/public/`.

| Удалить? | Что | Описание |
|----------|-----|----------|
| ☐ | Корневая `public/` | Папка в корне репозитория (рядом с index.html). |

**Критично:**

- Если у вас **есть** `wp-content/themes/oor-theme/public/` с полной копией медиа — корневую `public/` можно удалить (дубликат).
- Если **нет** папки `public` внутри темы и тема как-то использует корневую `public/` (симлинк, общий путь и т.п.) — корневую `public/` удалять **нельзя**, иначе сломаются картинки/шрифты/видео на сайте.

**Рекомендация:** перед удалением проверьте наличие и содержимое `wp-content/themes/oor-theme/public/`. Удалять только если медиа в теме полные и актуальные.

## 8. Дефолтные темы WordPress (оставить одну)

В репозитории лежат три стандартные темы; WordPress требует хотя бы одну активную тему помимо вашей (для переключения/безопасности). Удаление лишних уменьшит размер.

| Удалить? | Что | Описание |
|----------|-----|----------|
| ☐ | `wp-content/themes/twentytwentythree/` | Одна из стандартных тем. |
| ☐ | `wp-content/themes/twentytwentyfour/` | Одна из стандартных тем. |
| ☐ | `wp-content/themes/twentytwentyfive/` | Одна из стандартных тем. |

**Рекомендация:** удалить две из трёх (например, twentytwentythree и twentytwentyfour), оставив одну как fallback. В админке не переключайте сайт на удалённую тему.

**Рекомендация:** удалять только те скрипты, которые точно не будете запускать. При сомнениях — оставить.
